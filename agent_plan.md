# 에이전트 그래프
에이전트의 경우, 내부에서 스킬을 사용한 FSM을 추가로 제작할 수 있어야 함.
즉, 브레드 크럼브 방식의 서브 그래프가 되어야 한다.
이는 서브에이전트가 별도의 컨텍스트에서 돈다는 느낌을 유저에게 주기 때문에 중요하다
서브에이전트 그래프는 시작, 끝이 있음. 
끝은 여러 개 존재할 수 있으며, 각각의 끝은 노드 그래프에서 아웃풋이 됨
끝은 어떤 경위로든 시작으로부터 연결되어 있어야 함.
즉, 시작은 1개, 끝은 여러 개 일 수 있음.

# 참조 스킬
기존 Declarative 스킬은 전역 배경지식이라면, '재사용 가능한 참고용 노드'가 필요함.
기존 노드(프로시주얼 스킬, 에이전트)의 하단에 새로운 포트를 생성, 즉 상/하 방향 연결임.
참조 스킬은 위에 포트가 있음. 프로시주얼 스킬/에이전트 아래에 참조 스킬들이 달리는 형태.
