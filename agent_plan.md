# 에이전트 그래프
에이전트의 경우, 내부에서 스킬을 사용한 FSM을 추가로 제작할 수 있어야 함.
즉, 브레드 크럼브 방식의 서브 그래프가 되어야 한다.
이는 서브에이전트가 별도의 컨텍스트에서 돈다는 느낌을 유저에게 주기 때문에 중요하다
서브에이전트 그래프는 시작, 끝이 있음. 
끝은 여러 개 존재할 수 있으며, 각각의 끝은 노드 그래프에서 아웃풋이 됨
끝은 어떤 경위로든 시작으로부터 연결되어 있어야 함.
즉, 시작은 1개, 끝은 여러 개 일 수 있음.

> ❌ 미구현

# 엣지 스킬
상태변화 중에 뭔가 해야 할 때, 엣지를 단순 엣지로 쓰지 않고 Transfer Skill로 사용.
엣지 스킬은 특정 경우에 제한적으로 사용되기 때문에, 입출력 경로가 1개로 고정임. 일종의 일방통행 상태로 볼 수 있음. 

> ✅ 구현 완료 (2026-04-13)
> - `TransferSkill` 모델, `Transition.skill_ref`, `FIELD_REGISTRY` 기반 프론트매터
> - 레지스트리 ⚡ TRANSFER 섹션, SkillEditor 지원
> - 엣지 우클릭 → 스킬 설정/생성/해제 (undo 가능)
> - 엣지 라벨 렌더링 (`⚡ name`), Transfer 색상/두께 인디케이터



# 순환 연결
워크플로우를 짜는 것이니 만큼, 순환 연결이 가능해야 함. 

> ⚠️ 부분 구현
> - A→B→A 방향 순환은 이미 동작 (일반 전이로 연결 가능)
> - 셀프루프(A→A)는 현재 차단 중 (`target is not self._connect_source` 조건)
> - 셀프루프 허용 여부 결정 필요

# 프로퍼티 창의 문제
현 상태에서 스킬, 에이전트는 더블클릭 후 별도 탭에서 편집하고 있음. 프로퍼티 창이 가지는 의미가 거의 없는 상황.

> ❌ 미구현

# UI 개선
